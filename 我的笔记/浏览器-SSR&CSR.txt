
一、SSR（Server Side Rendering）服务端渲染 是一种将网页内容在服务器端生成并发送到客户端的渲染方式。
相对于传统的客户端渲染（Client Side Rendering），SSR 在服务器端生成完整的 HTML 页面，然后将其发送到浏览器显示给用户。



二、CSR：客户端渲染
在传统的 客户端渲染 中，浏览器发送请求到服务器，服务器返回一个空白的 HTML 文件，浏览器处于白屏状态。
浏览器再次发送多个请求到服务器，分别请求 css 、图片、js ，服务器响应对应资源，然后浏览器开始执行 JavaScript 来生成和渲染页面内容。
这种方式下，初始页面加载时只有一个基本的 HTML 骨架，然后通过异步加载和执行 JavaScript 来填充内容。
这样的过程可能导致一些页面内容在初始加载时的延迟和白屏现象，不利于 SEO。

SSR：服务端渲染
而使用 服务端渲染，浏览器发送请求到服务器，服务器返回一个 HTML 文件，包括完整的数据和 HTML 内容，浏览器显示出页面。
浏览器再次发送多个请求到服务器，分别请求 css 、图片、js ，服务器响应对应资源，然后浏览器开始执行 JavaScript ，浏览器直接展示这个完整的页面给用户。
这样用户在初始加载时就能立即看到完整的页面内容，无需等待异步加载和执行 JavaScript，有利于 SEO。



三、SSR 渲染的优点包括：
	更好的首次加载性能：由于服务器端已经生成了完整的 HTML 页面，用户在初始加载时能够立即看到页面内容，减少白屏时间和延迟。
	更好的 SEO（搜索引擎优化）：搜索引擎可以直接从服务器端获取到完整的 HTML 页面，提高了搜索引擎对页面内容的识别和索引能力。
	更好的用户体验：用户在初始加载时能够立即看到内容，不需要等待异步加载和执行 JavaScript 的过程。

需要注意的是，SSR渲染在服务器端进行了大部分的计算和渲染工作，可能会增加服务器的负载和响应时间。同时，开发和维护SSR应用程序也相对复杂一些，因为需要考虑服务器端和客户端的代码逻辑。
